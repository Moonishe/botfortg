# Max Mode (5 candidates + judge + replay)

Ты выполняешь Max Mode — трёхфазный протокол для critical-задач.

## Фаза 1: Propose (5 параллельных кандидатов)

Запусти 5 кандидатов ОДНОВРЕМЕННО:

```
task(subagent_type="worker", permission={edit: deny, bash: deny}, prompt=...)
```

Каждый кандидат:
- **Read-only** (schema-only tools): read, grep, glob, search, но НЕ edit/bash/write
- Возвращает **текстовое предложение** (план, дифф, описание решения)

### 5 стратегий для кандидатов

**Candidate 1 — Minimalist:** Минимальное изменение. Только необходимое. Приоритет: не сломать существующее.

**Candidate 2 — Clean Architecture:** Правильное решение по паттернам. Даже если больше кода. Приоритет: maintainability.

**Candidate 3 — Performance-first:** Оптимальное по скорости/памяти. Приоритет: производительность.

**Candidate 4 — Defensive:** Максимально безопасное. Валидация всего, обработка всех ошибок. Приоритет: security + reliability.

**Candidate 5 — Creative:** Нестандартный подход. Может использовать неочевидные паттерны. Приоритет: инновация.

## Фаза 2: Judge

Ты (главный агент) читаешь все 5 предложений и выбираешь лучшее.

**Критерии:**
1. **Correctness** — решает ли проблему правильно?
2. **Elegance** — насколько чистое и понятное решение?
3. **Completeness** — все ли аспекты покрыты?
4. **Minimality** — не больше изменений чем нужно?
5. **Risk** — какой риск регрессии?

## Фаза 3: Replay

Победившее предложение отдаётся новому worker'у с execute:

```
task(subagent_type="worker", prompt="Реализуй это предложение: <победитель>")
```

После реализации → стандартный D5→R5 pipeline.

## Правила
- Только для critical/complex задач
- Не для simple багов
- 5 кандидатов → параллельно
- Проигравшие предложения — overhead (токены), но не в коде
